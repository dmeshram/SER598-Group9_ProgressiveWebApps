History of Progressive Web Apps

A brief history of PWAs, the web standards that enabled them, and the milestones that made them mainstream.

Introduction

Progressive Web Apps (PWAs) are a set of web-platform features combined to deliver app-like experiences on the web. The PWA concept grew from years of incremental improvements in browser capabilities, networking, and developer tooling.

Timeline & Key Milestones

  • 2007–2010: Mobile web and app stores begin — developers seek better mobile UX.
  • 2012: Service workers concept begins to appear in specs and experiments.
  • 2014: App-like web experiences multiply; early offline patterns emerge.
  • 2015: Service Workers shipped in Chrome; offline and push become viable.
  • 2015–2016: Web App Manifest and installability get standardized.
  • 2017: PWAs promoted widely at conferences; major sites adopt installable experiences.
  • 2018–2020: Broader browser support and improved power/feature parity.

Why PWAs Matter

PWAs lowered the friction for delivering engaging, offline-capable applications without the overhead of native app stores. They brought features like background sync, push notifications, and installability to the web platform, enabling better user retention and more resilient web apps.

Browser Support & Adoption

Browser support has historically been staggered: Chrome and Chromium-based browsers led with service workers and manifests, while other vendors added support later. Today most modern browsers support the core PWA features, but subtle differences in APIs and platform integration remain.
