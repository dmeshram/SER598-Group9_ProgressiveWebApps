Activities ‚Äî how to use this page

This page contains short labs and a micro-quiz to reinforce the implementation concepts. Each lab includes objectives, steps, and a "Mark complete" button. Your progress is saved locally in the browser.

Lab 1: Create a Tiny manifest.json

Objective: create a minimal web app manifest and preview it in DevTools ‚Üí Application ‚Üí Manifest.

Steps

  1. Create a file named manifest.json in your frontend's public folder.
  2. Paste the sample manifest below and adjust names/icons as needed.
  3. Open your site in Chrome, then open DevTools ‚Üí Application ‚Üí Manifest to verify fields.
{
  "name": "My PWA Demo",
  "short_name": "PWA Demo",
  "start_url": ".",
  "display": "standalone",
  "background_color": "#ffffff",
  "theme_color": "#0066cc",
  "icons": [
    { "src": "/icons/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icons/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}

Success criteria: Manifest loads in DevTools and shows app name, icons, and start_url.

Lab 2: Add a Basic Service Worker (Offline)

Objective: register a simple service worker that precaches static assets and serves an offline page.

Steps

  1. Create service-worker.js in your frontend public root.
  2. Paste the sample below, update the cache list to match your assets, and register it from your app.
  3. Open DevTools ‚Üí Application ‚Üí Service Workers to verify registration and try "Offline" mode. 
// service-worker.js (simple precache)
const CACHE_NAME = 'pwa-example-v1';
const PRECACHE_URLS = [
  '/',
  '/index.html',
  '/offline.html',
  '/css/styles.css',
  '/js/main.js'
];

self.addEventListener('install', event => {
  event.waitUntil(
    caches.open(CACHE_NAME).then(cache => cache.addAll(PRECACHE_URLS))
  );
});

self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request).then(resp => resp || fetch(event.request))
      .catch(() => caches.match('/offline.html'))
  );
});

Success criteria: Assets served from cache when offline; offline.html shown if navigation fails.

Lab 3: Lighthouse Audit & Improvements

Objective: run Lighthouse PWA audit and apply two quick improvements (icons, manifest, or caching).

Steps

  1. Open Chrome, load your app, open DevTools ‚Üí Lighthouse.
  2. Run the PWA audit and note failing checks.
  3. Fix one issue (e.g., missing icons) and re-run the audit to show improvement.

Success criteria: PWA score increases after applying changes.
