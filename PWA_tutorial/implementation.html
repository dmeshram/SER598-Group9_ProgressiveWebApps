Overview of implementation

This project uses a two-tier architecture:

  • Backend: Spring Boot (Maven) ‚Äî provides REST APIs used by the frontend.
  • Frontend: Modern JS app (Node/npm) ‚Äî serves the UI and PWA assets (manifest, service worker sample).

Below are the exact commands to run each part locally.

Prerequisites

  • Java 11+ (or the Java version specified in your `pom.xml`). Confirm with java -version.
  • Maven (3.6+) ‚Äî used to build and run the Spring Boot backend. Confirm with mvn -v.
  • Node.js (14+) and npm ‚Äî used for frontend toolchain. Confirm with node -v and npm -v.
  • Git ‚Äî to clone the repository.

Run backend (Spring Boot)

From the repository root, change into the backend module (if applicable) and run:

mvn spring-boot:run

Notes:

  • If your backend is in a subfolder (for example `server/` or `backend/`), run the command from that folder.
  • To build a production JAR: mvn clean package then run: java -jar target/your-app.jar.
  • Adjust JVM memory or profiles with environment or Maven properties as needed (e.g. -Dspring.profiles.active=dev).

Run frontend (Node / npm)

From the frontend folder (commonly `frontend/` or `client/`), run the usual install & dev commands:

npm install
npm run dev

Typical notes:

  • If the frontend expects a specific API base URL, set the environment variable or config (example: VITE_API_BASE_URL, REACT_APP_API_URL, or a `.env` file; check your frontend README).
  • Default dev server ports: frontend often runs on http://localhost:3000 or 5173 (Vite). Backend often uses http://localhost:8080. Adjust proxy or CORS settings accordingly.
  • For production builds, run npm run build and serve the static output (or integrate with backend).
